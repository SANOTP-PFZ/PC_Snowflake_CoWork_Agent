## Context

The project folder is empty except for `Wireframe.html`. The wireframe defines a 4-screen single-page app for discovering and launching Cortex agents. I'll translate it into a multi-page Streamlit app with session-state-based navigation.

## Implementation Steps

### 1. Create `app.py` (main entry point)
- Set page config (wide layout, title "Cortex Agent Hub")
- Inject custom CSS to match the wireframe's design tokens (warm neutrals, rounded cards, chip styles)
- Render the top header bar (logo, title, user avatar)
- Use `st.session_state["screen"]` to control which screen is shown (landing, ta, tad, agent)
- Import and call the appropriate screen renderer

### 2. Create `pages/` helper modules (or sections within `app.py`)
Since Streamlit's native multi-page routing doesn't support passing state cleanly between pages, I recommend a **single `app.py` with screen functions** organized in a `screens/` package:

- `screens/__init__.py`
- `screens/landing.py` — Brand performance strip, category cards, stats tiles
- `screens/ta_grid.py` — TA agent listing with brand filter chips
- `screens/tad_grid.py` — TA+Data agent listing with brand + data source filters
- `screens/agent_detail.py` — Sidebar metadata + mock chat interface

### 3. Create `data/agents.py` (static agent data)
- `BRANDS` list with name, category, trend %, direction
- `TA_AGENTS` list (name, brand tag)
- `TAD_AGENTS` list (name, brand tag, data source tag)

### 4. Create `components/` reusable UI helpers
- `components/__init__.py`
- `components/header.py` — Top navigation bar
- `components/cards.py` — Agent card renderer, brand mini-card, stat tile
- `components/filters.py` — Chip-based filter row using `st.pills` or custom buttons
- `components/chat.py` — Mock chat panel with message bubbles and input

### 5. Create `styles.py` or embed CSS
- Custom CSS injected via `st.markdown(unsafe_allow_html=True)` to style cards, chips, chat bubbles, and layout to approximate the wireframe look

### 6. Create `requirements.txt`
- `streamlit>=1.35`
- `plotly` (optional, for sparkline charts in brand cards — or use `st.line_chart` with minimal config)

## Key Design Decisions

| Concern | Approach |
|---------|----------|
| Navigation | `st.session_state["screen"]` + callback buttons (no native multi-page) |
| Filters | `st.pills()` (Streamlit 1.35+) for chip-like selectors |
| Card grid | `st.columns(3)` with `st.container` per card |
| Brand sparklines | `st.line_chart` or tiny Plotly figure in each column |
| Chat UI | `st.chat_message` for bubbles + `st.chat_input` for the input box |
| Styling | Injected CSS via `st.markdown` to match wireframe colors/spacing |
